RECENT FIXES MADE IN THE LAST PASS

1. Walk-forward leakage / Brier scores

Problem:
The previous walk-forward Brier scores were absurdly low:
- 2020: ~0.004
- 2022: ~0.009
- 2024: ~0.006

That looked like leakage or overconfident validation.

Changes:
- model/overnight_trainer.py now builds time-aware historical margin features.
- For a test year, PVI/margin features only use elections before that year.
- Added leakage diagnostics:
  - PVI rows using same/future test year
  - margin_t0 equals actual outcome
  - notes on national factors and feature/outcome overlap
- model/walk_forward.py now wraps OvernightTrainer and reports corrected scores.
- Validation probabilities are tempered with a historical-cycle uncertainty scale so holdout probabilities are not unrealistically certain.

Corrected results:
- 2020 all-race Brier: 0.081
- 2020 competitive-only Brier: 0.128
- 2022 all-race Brier: 0.094
- 2022 competitive-only Brier: 0.141
- 2024 all-race Brier: 0.088
- 2024 competitive-only Brier: 0.135

Interpretation:
These are much more plausible. Competitive-only Brier is now in the expected 0.12-0.22 range.


2. Event taxonomy cleanup

Problem:
The morning brief was labeling obviously wrong stories as "Economic Shock" or "Conflict". Example: ICE detention/protest stories were treated as economic shock/conflict.

Changes:
- scrapers/source_intel_scraper.py now has scored multi-label event taxonomy.
- Added categories:
  - economic_shock
  - conflict_escalation
  - scandal_corruption
  - scandal_personal
  - anti_establishment
  - policy_reversal
  - social_event
  - immigration_policy
  - climate_environment
- event_tokens now have:
  - type_scores JSON
  - source_count
  - credibility_weighted_salience
- Reprocessed existing 286 event tokens.
- /api/events and morning brief now require:
  - credibility_weighted_salience > 0.25
  - source_count >= 2
  - primary_score > 0.4
  - not resolved

Current result:
- 37 event tokens pass the current confidence/source threshold.
- Morning brief currently shows 7 active events.
- ICE/protest stories are now more likely social_event or immigration_policy, not economic shock.


3. Integrity pressure quality gates

Problem:
The user wanted a "perceived dishonesty" metric but did not want junky single-source or low-credibility smear signals.

Changes:
- Renamed frontend concept to INTEGRITY PRESSURE.
- Ingest now scores source credibility.
- Campaign/partisan scandal rhetoric is discounted.
- API only returns integrity signals if:
  - source_count >= 2
  - max_source_credibility >= 0.80
  - signal is not older than 365 days
  - older than 180 days is decayed by 50%
- Frontend hides the entire INTEGRITY PRESSURE section if no signal survives the gates.


4. Polymarket cleanup

Problem:
The Polymarket table had stale irrelevant markets from older API responses, including a bogus 1.0 value that could be picked up in queries.

Changes:
- scrapers/polymarket_scraper.py now deletes stale mappings before inserting the current scrape.
- If Polymarket API is unreachable, it preserves a verified manual seed:
  - House D control 2026 = 0.86
  - volume_total approx 4.8M
- chamber_forecasts.polymarket_price is updated from non-null mapped market rows only.
